# Part 1 — Frontend Analysis Report

Generated by walking the React app under `frontend/src/` and cross-referencing with the existing Express backend in `backend/`.

---

## Stack

- **Framework:** React 18 + Vite
- **Routing:** `react-router-dom` v6
- **State:** Redux Toolkit + `redux-persist` (localStorage) — slices: `auth`, `job`, `company`, `application`
- **HTTP client:** `axios` with `withCredentials: true` (cookie-based auth)
- **UI:** Tailwind CSS + shadcn/ui (Radix primitives)
- **i18n:** `i18next` + `react-i18next` (en, ja, hi, fr, ar — see `src/locales/`)
- **Forms / Toasts:** native + `sonner`
- **Notifications:** `sonner`

---

## Authentication method

**JWT in httpOnly cookie**, set by the backend on `POST /api/v1/user/login`. The frontend always sends `withCredentials: true`. There is no Authorization header; no token in `localStorage`. Logout = `GET /api/v1/user/logout` which clears the cookie.

> ⚠️ Pre-fix state: `frontend/src/components/auth/Login.jsx` used `MOCK_USER` from `utils/mockData.js` and never hit the backend. **This has been wired to the real `/user/login` endpoint.** `Signup.jsx` was already wired correctly.

## Issues found, NOT fixed (deliberate, listed for follow-up)

- **No role-based middleware on recruiter-only endpoints** (e.g. `POST /job/post`, `POST /company/register`, `POST /application/status/:id/update`). Currently any authenticated user can call them; the frontend gates by role only. Add an `isRecruiter` middleware.
- **`GET /application/apply/:id` uses GET for a side-effecting action**. Frontend uses GET; changing it to POST would require a frontend update too. Leaving as-is for backward compat; mark for cleanup.
- **No input validation library** (Zod / Joi). All controllers do ad-hoc `if (!field)` checks. Adding Zod schemas is a recommended next step.
- **Multer accepts any file type / size**. Add `fileFilter` and `limits`.
- **Several controllers swallow errors** (`} catch (error) { console.log(error); }`). They should `next(err)` so the global error handler can respond. Mechanical fix; listed as cleanup.
- **`profile.resumes` deletion compares by `_id`** — works for Mongo subdocs but not validated against ObjectId format.
